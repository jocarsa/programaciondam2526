A partir de este momento:

Vosotros ya teniais la obligación de usar GitHub para gestionar
el código

Tenéis también el deber de crear commits periódicos en los que se vea
el crecimiento del proyecto
(Resulta sospechoso presentarse el dia del examen con un 
unico commit realizado una hora antes del examen)

Además tenéis el deber de, en este mes que queda, crear ramas
y fusionarlas

El objetivo es que:
1.-Cuando tengas el programa funcionando
2.-Quieras meter una nueva funcionalidad
3.-No estés seguro de si la vas a cagar:
4.-Crea una rama
5.-Si la cagas la destruyes
6.-Si no la cagas, la fusionas

Dentro de un mes, en los exámenes, revisaré que en vuestro proyecto haya
1.-commits
2.-ramas
